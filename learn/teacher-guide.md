# Teacher Guide

## Purpose

This file is for the person teaching the student.

The goal is not to finish pages quickly.
The goal is to make sure the student understands the system clearly and can explain it in simple language.

## Teaching Style

Use these rules every day:

- explain slowly
- use simple words first
- do not assume hidden knowledge
- ask the student to repeat in her own words
- connect every concept to this project
- teach the problem first, then the solution
- show why a design choice exists

## Golden Teaching Formula

For every topic, explain in this order:

1. what it is
2. why it exists
3. where it fits in the project
4. what problem it solves
5. what breaks if we remove it

## Daily 4-Hour Structure

### Hour 1

- explain concept
- define important words
- give one real-world analogy

### Hour 2

- explain diagram
- explain request flow
- ask the student to redraw the flow

### Hour 3

- open repo files
- connect concept to code
- explain one file line by line if needed

### Hour 4

- do exercise
- ask mini interview questions
- ask the student to teach the topic back

## What To Check Every Day

At the end of the day ask:

- Can she explain it without reading?
- Can she connect concept to code?
- Can she say why this design exists?
- Can she answer one small interview question?

## When The Student Is Confused

If she gets confused:

- stop using big words
- use one box-and-arrow explanation
- use one simple example
- compare it with real life
- ask only one question at a time

## Biggest Danger

Do not let her memorize cloud words without understanding flow.

She must understand:

- request
- validation
- storage
- processing
- response

These five ideas are the foundation of this project.

## Teacher Tip For Interviews

Make her answer in:

- 1 sentence
- 3 sentences
- 1 minute

This builds real confidence.
