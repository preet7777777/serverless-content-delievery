# Student Progress Tracker

## How To Use This File

Use this as a checklist.

Mark each item only when the student can explain it in her own words.

## Foundation

- [ ] I can explain what a backend is.
- [ ] I can explain what an API is.
- [ ] I can explain request and response.
- [ ] I can explain authentication and authorization.

## Architecture

- [ ] I can explain the full project flow.
- [ ] I can explain control plane and data plane.
- [ ] I can explain why direct upload is used.
- [ ] I can explain why processing is asynchronous.

## Backend

- [ ] I can explain what `AppConfig` does.
- [ ] I can explain what request and response models do.
- [ ] I can explain what validation does.
- [ ] I can explain what `UploadAuthorizationService` does.
- [ ] I can explain what `ImageProcessingService` does.
- [ ] I can explain why handlers should stay thin.

## Storage And Processing

- [ ] I can explain raw bucket vs optimized bucket.
- [ ] I can explain pre-signed URL.
- [ ] I can explain what the image processor creates.

## Local Testing

- [ ] I can explain the local Docker flow.
- [ ] I can explain what LocalStack is doing.
- [ ] I can explain why the local runner exists.

## Interview

- [ ] I can explain the project in 30 seconds.
- [ ] I can explain the project in 1 minute.
- [ ] I can answer why not to upload through backend.
- [ ] I can answer what Cognito does.
- [ ] I can answer how to debug a missing image.

## Build Confidence

- [ ] I can describe which file to create first in a scratch build.
- [ ] I can explain what each main backend file should do.
- [ ] I can explain how the browser and backend connect.
- [ ] I can explain the project without reading notes.

## Teacher Sign-Off

- [ ] Student understands the system flow.
- [ ] Student understands backend responsibilities.
- [ ] Student can explain the project calmly.
- [ ] Student is ready for intern-level discussion.
